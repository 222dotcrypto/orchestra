# Anti-patterns — что НЕ работает

> Обновляй после каждой задачи с rework>0 или failed. Один инсайт = одна строка. Удаляй устаревшее.

## Декомпозиция
- Субъективные criteria ("чистый код", "хорошая архитектура") → всегда rework, невозможно проверить бинарно
- Задача без constraints → воркер рефакторит код который не просили трогать
- Одна задача на >10 файлов → воркер теряет контекст, результат неполный

## Промпты
- Передача всего контекста проекта воркеру → шум, воркер не понимает что главное
- description без конкретики ("сделай хорошо", "оптимизируй") → воркер додумывает, додумывает неправильно
- Промпт через --system-prompt с экранированием → теряется форматирование, ломаются спецсимволы

## Координация
- Два воркера на одном файле → семантические конфликты, невозможно merge
- Запуск зависимой задачи до ревью предыдущей → rework каскадом
- Передача полного результата между фазами → информационный перегруз

## Ревью
- >2 rework одной задачи → проблема в декомпозиции, не в воркере
- Rework-комментарий "сделай лучше" без конкретики → тот же результат
- Принять "потому что устал ревьюить" → технический долг

## Ресурсы
- Воркер на тривиальную задачу (5 строк) → 50x overhead по токенам
- Мониторинг каждые 5 секунд → съедает контекст Мозга на чтение JSON
- Запуск 6+ воркеров на simple задачу → overhead координации > экономия параллелизма

## Инфраструктура (уроки прогона #1)
- Heartbeat через jq каждые 30 сек = 300+ лишних tool calls. Не нужен — есть tmux/ps проверка
- Промпт 50 строк (90% boilerplate) = лишние токены на каждого воркера. Протокол → файл
- Polling JSON-файлов каждые 15 сек = ~35 циклов × 3-6 reads = основной пожиратель токенов Мозга
- Запуск зависимых воркеров в фазе 1 → воркер поллит несуществующие файлы, зависает, нужен restart
- CLAUDECODE env var наследуется → nested session crash. Всегда unset перед exec claude
- allowedTools без конкретных bash-команд + pipe mode = воркер не может запустить тесты
- Большой промпт (8KB) целиком через `-p` → claude молча не выдаёт результат. Правильно: инструкции → `--system-prompt`, конкретная задача → `-p` (1-2 предложения)
