Проверь результат последнего code review.

## Алгоритм

### 1. Найди последний результат

Выполни `ls -t .brain/review/results/review-*.md | head -1` чтобы найти самый свежий файл результата.

Если файлов нет — сообщи "Результатов ревью пока нет. Запусти `/review` чтобы начать проверку."

### 2. Проверь статус ревьюера

Выполни `tmux list-windows -t orchestra -F '#{window_name}' 2>/dev/null | grep -c reviewer` чтобы понять, работает ли ревьюер.

- Если окно reviewer существует — скажи "Ревьюер ещё работает. Подожди немного."
- Если окна нет — покажи результат.

### 3. Покажи результат

Прочитай файл результата и покажи пользователю:
- Вердикт (ПРИНЯТО / С ЗАМЕЧАНИЯМИ / НА ДОРАБОТКУ)
- Критичные проблемы (если есть)
- Важные замечания (если есть)
- Саммари для основного агента

### 4. Предложи действия

Если вердикт "С ЗАМЕЧАНИЯМИ" или "НА ДОРАБОТКУ" — спроси:
1. Применить все фиксы автоматически?
2. Применить только критичные?
3. Показать детали конкретного замечания?
4. Пропустить (оставить как есть)?

Если пользователь выбрал "применить фиксы" — прочитай секцию "Как исправить" для каждого замечания и примени изменения в соответствующих файлах.
