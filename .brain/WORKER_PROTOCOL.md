# Протокол воркера Orchestra

## Начало работы
1. Прочитай `.brain/tasks/` — найди задачу с `"assigned_to": "{WORKER_ID}"` и `"status": "assigned"`
2. Обнови задачу: `"status": "in_progress"`
3. Обнови `.brain/workers/{WORKER_ID}.json`: `"status": "busy"`, `"current_task": "task-XXX"`

## Выполнение
- Прочитай `description` задачи — там КОНТЕКСТ, ЗАДАЧА, ОГРАНИЧЕНИЯ
- Прочитай `acceptance_criteria` — это чек-лист готовности
- Меняй ТОЛЬКО файлы из `owned_files`
- Результат запиши в файл из `result_path`

## Завершение

### Успех
1. Обнови задачу: `"status": "done"`
2. Создай сигнальный файл:
```bash
touch .brain/signals/{TASK_ID}.done
```
3. **Заверши сессию** — не жди дальнейших указаний

### Неудача
Если задача невыполнима (ошибка компиляции, недостающие зависимости, невозможно выполнить требования):
1. Обнови задачу: `"status": "failed"`
2. Запиши причину в `.brain/logs/{WORKER_ID}.log`
3. Создай сигнальный файл:
```bash
touch .brain/signals/{TASK_ID}.failed
```
4. **Заверши сессию**

## Самопроверка перед .done

**ОБЯЗАТЕЛЬНО** перед созданием `.done` сигнала:

1. Перечитай `acceptance_criteria` из задачи
2. Проверь каждый критерий — бинарно (да/нет)
3. Проверь что `result_path` существует и не пустой
4. Для кода — проверь что нет синтаксических ошибок:
   - Python: `python3 -c "compile(open('file.py').read(), 'file.py', 'exec')"`
   - JS: `node --check file.js`
   - Bash: `bash -n file.sh`
5. Проверь что созданные файлы содержат то, что ожидается (не заглушки, не `pass`, не `TODO`)
6. Если хотя бы один критерий не пройден — исправь ПЕРЕД `.done`

**КРИТИЧНО**: runner проверяет результат автоматически (acceptance_signals + suspicious detection). Если отправишь `.done` с невалидным результатом — задача будет отклонена и перезапущена. Лучше потратить 30 секунд на самопроверку, чем получить rework.

## Правила
- НЕ трогай файлы вне `owned_files`
- НЕ меняй чужие задачи
- Результат ТОЛЬКО в свой `result_path`
- Если застрял — `"status": "failed"` + `.failed` сигнал + описание в логе
- Атомарные записи JSON: write → `/tmp/file.tmp` → `mv` в целевой путь
- После создания сигнала (.done или .failed) — ЗАВЕРШАЙ сессию
